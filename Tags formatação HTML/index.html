    Módulo 4: Características do CSS

    1. Herança: É quando a propriedade de uma class é herdada por todas as tags existentes dentro daquela class, existem algumas propriedades que não são herdadas.

    2. Especificidade: É o ato de especificar um elemento do html para o css, quanto mais específico melhor, uma class sempre terá prioridade em relação ao uma tag html (div etc.), o html aplica estilo de cima para baixo.

    3. Cascata: A ordem de leitura do navegador se dar da seguinte forma: primeiro ele ler o arquivo css, depois ler o arquivo principal, o index html de cima para baixo, assim, embora tenha um estilo para determinada tag ou class no documento css, se houver um outro estilo no html, esse tem prioridade na execução.

    4. Unidade de medida absoluta: São unidades que não são alteradas de acordo com a estrutura que tem ao redor, não mudam de acordo com as especificações do dispositivo, ex. px, pt, in, cm, mm.

    5. Unidade de medida relativa a fonte: Elas mudam de tamanho baseadas nos elementos da página.
      1. EM o tamanho em é baseado no tamanho font-size do elemento pai. 1 em = 10 px.
      2. REM Não se baseia no elemento pai, como o em, mas no primeiro elemento, no caso o elemento html, quando não se especifica o tamanho do html, os navegadores colocam por padrão o valor 16px.
      3. Porcentagem: A porcentagem é relacionada ao tamanho do elemento pai. Não se usa muito para fontes, usa-se mais para estruturas.
      4. EX: O ex acompanha o tamanho da letra "x" da fonte que ta sendo usada, se não tiver nenhuma fonte especificada, o navegador usa sua fonte padrão.
      5. CH: O tamanho acompanha o espaço usado pela quantidade de caracteres do que estiver escrito, só funciona quando é usada uma fonte monospace(quando todos os caracteres tem o mesmo tamanho).
      6. VIEWPORT: São unidades de medidas relativas ao espaço vizível ao usuário, relativo ao tamanho da tela, para usa-la pode-se usar dois elementos:
        1. VH: Atualiza o elemento de acordo com a altura do elemento pai.
        2. VW: Atualiza o elemento de acordo com a largura do elemento pai.
        3. VMIN: Verifica qual o menor elemento do site, se é altura ou largura e atualiza o elemento de acordo com ele.
        4. VMAX: Verifica qual o maior elemento do site, se é altura ou largura e atualiza o elemento de acordo com ele.
    6. Inherit: Valor usado para forçar uma herança, normalmentes são usados para tamanhos de fontes e cores.

    Módulo 5: Responsividade:

    1. Mobile firsh: Na tradução livre é o celular primeiro. Os sites se adequarão ao tamanho da tela que o usuário estar usando, assim se cria o site baseado primeiramente para o celular e após isso adapta para telas maiores. No entanto hoje em dia se estar agindo de forma meio que contrária, pois cria-se o site baseado na tela maior acompanhando o mesmo para telas menores.

    2. Layout normal x layout adaptativo x layout responsivo:
      1. Layout normal: O site não se adapta para outros tamanhos de tela, qundo se abre em uma tela menor, o navegador cria uma barra de rolagem para percorrer o retante do site que não aparece na tela.
      2. Layout responsivo: O site é criado de uma forma que o tamanho do mesmo va se adptando ao tamanho do site vá gradualmente se moldando ao tamanho da tela.
      3. Layout adaptativo: O site é criado de forma que o mesmo se adapte ao tamanho da tela através de tamanhos pre-estabelecidos para determinada tela. Os tamanhos mais usados no adaptativo são: 320, 480, 760, 960, 1200 e 1600.
      Normalmente se usa mais uma junção do layout responsivo com o adaptativo.

    3. Viewport:É uma função que se usa no head que faz a largura do site se adaptar ao tamanho da tela do usuário. o código é o a seguir:
      < meta id="viewport" name="viewport" content="width=device-width, user-scalable=no">

    5. @media (css): É basicamente um css dentro do outro css aplicado quando se atinge determinadas condições estabelecidas de responsividade, é o recurso mais utilizado para responsividade de sites. Através do @media se pode aplicar características específicas para diversas situações, exemplo: o @media only print, aplica condições para impressão, o @media only scren aplica condições apenas para tela, sem aparecer na impressão. Quando não se usa uma condição e se deseja aplicar algo a tudo, se utiliza o @media all.
      1. Orientation no @media: A propriedade orientation em @media aplica determinada características no site de acordo com a orientação que estiver a tela (retrato ou paisagem), o navegador se baseia nas dimenções de largura e altura para determinar se a tela estar no modo retrato ou paisagem, o comando @media(orientation: landscape) aplica a característica para quando a tela tiver no modo paisagem, já o comando @media (orientation:portrait) aplica a característica para quando a tela estiver em modo retrato.

      2. Aspect-ratio no @media:Não sei explicar direito, mas é basicamente outro condicionador de responsividade, que altera determinada características da propriedade de acordo com regras pre-definidas. O código é @media (aspect-ratio: 1/1){aplicação} no caso desse exemplo, ele aplica a característica quando a proporção da tela atingi 1 para altura e 1 para largura, essa proporção pode ser alterada, bem como pode ser usada um @media (min-aspect-radio: 2/1) por exemplo ou ainda um @media (max-aspect-ratio: x/x).
    6. Função var: É usada para se criar uma variável em css para determinado atributo, ex. uma cor específica para ser usada em diversos elementos. A vantagem é que, caso se deseje alterar a cor de diversos elementos, não se precisa alterar manualmente de um por um, basta criar uma variável cor para todos os elementos e ai pode se mexer apenas na cor da variável que os elementos todos mudam automaticamente

      • Criar a variável:
        • No topo do documento se cria da seguinte forma, para que possa ser usada em todo o documento: :root{--nome da variável:valor da variável;}
          Ex.:root{
          --bg-color: black;
          }

        • Em uma div específica para usar os valores nos elementos daquela determinada div: --nome da variável:valor da variável;
          Ex. .nome da div{
          --Azul: blue;
          }

      • Usar as variáveis criadas:
        • Em um elemento que se deseje usar a variável criada no topo, se usa o códogo da seguinte maneira: var(--nome da variável);
          Ex. background-color: var(--bg-color);

        • Para usar as variáveis locais (criadas e usadas dentro de uma mesma div): Basta colocar o elemento seguido do termo var e o nome da variável entre parênteses.
          Ex. color: var(--azul);

      Obs: para fins de uma melhor organização, se costuma usar a variável no root, no topo do documento css.

      Existe a opção de colocar um atributo ou uma outra variavel após uma que estar sendo usada para caso ela não funcione, a outra funcionar, veja exemplo abaixo:
      background-color: var(--bg-color, outra variável ou outra cor);
