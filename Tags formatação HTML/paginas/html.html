Algumas tags principais de html


  1. Header: Não tem nenhuma função vizual, mas ajudar ao motor de busca a saber que todo conteúdo dentro dela é um cabeçalho, um título, um sub-título, uma introdução da página. Tem a facilidade de se fazer estilizações no css usando apenas o nome da tag.

  2. Footer: Também não tem nenhuma função vizual no site, apenas serve para colocar o rodapé do site. Tem a facilidade de se fazer estilizações no css usando apenas o nome da tag.

  3. Nav: Também não tem nenhuma função vizual no site, no entanto, ela serve para dizer aos motores de buscas que seu conteúdo é uma espécie de menu principal do site. Tem a facilidade de se fazer estilizações no css usando apenas o nome da tag.

  4. Section: Serve para separar o site em algumas seções diferentes.

  5. Article: Diz para o motor de busca ou para o scaner que ali tem um conteúdo importante ou principal daquele site.

  6. Aside: É o contrário do article, o conteúdo é interpretado com conteúdo secundário ou menos importante do site.

  7. Time: O navegador entende como um conteúdo relacionado a data, nele pode-se usar o atributo datetime para informar qual data é aquela.

Todas as tags acima não apresentam nenhuma função vizual no site, elas servem para causar efeitos no buscador ou navegador ou motor de buscas etc. As mesmas são úteis para aplicar formatações no css usando-as sem que seja preciso criar class ou ids para aquele conteúdo.

  9. Validações nativas: São validações feitas nos formulários para se cetificar que o usuário vai responder aos campos de acodo com o pedido pelo mesmo. Existem as validações que podem feitas através do JS, no entanto essa que vamos aprender chama-se nativas por que usam recursos do navegador.
  10. Required: Esse recurso usado no campo do formulário, obriga ao usuário a preencher aquele campo antes de enviar o formulário. No exemplo de formulário acima há o recurso presente no campo "Primeiro nome".
  11. minlength: Esse recurso usado no campo do formulário, obriga ao usuário a usar uma quatidade mínima de caracteres naquele campo. No exemplo de formulário acima há o recurso presente no campo "Primeiro nome".
  12. maxlength: Esse recurso usado no campo do formulário, obriga ao usuário a usar uma quatidade máxima de caracteres naquele campo. No exemplo de formulário acima há o recurso presente no campo "Primeiro nome", onde não pode digitar mais de 5 caracteres.
  13. step: Esse recurso pode ser usado em um campo numerico do formulário, ele obriga ao usuário a aumentar a numeração de acordo com o intervalo exigido pelo programador. No exemplo de formulário acima há o recurso presente no campo "idade", onde o usuário do pode acrescentar números de 2 em 2.
  14. Max e Min: são recursos usados em campos numericos para se exigir a coleta e um número máximo e mínimo. No campo idade não se aceita números menores quem 18 nem maiores que 100. Há também a opção de se usar o max e min no type data, para isso se usa a data no padrão internacional no código, ex. AAAA-MM-DD.
  15. Hidden: É um campo do imput que não aparece para o usuário, no entanto levam informações para o banco de dados.
